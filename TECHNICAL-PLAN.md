# AI 命理小程序 - 技术方案

> 本文档详细记录 AI + 玄学 + 灵魂伴侣小程序的技术实现方案

---

## 一、项目概述

### 1.1 项目背景

基于海外爆款产品 Starla/Astra 的"AI 灵魂伴侣画像"概念，结合中国传统玄学（八字、星盘、塔罗），打造一款面向微信生态的情绪价值型小程序。

**海外验证数据：**
- Starla：上线半年，月收入 218 万美元，下载峰值日增 11 万
- 核心逻辑：为孤独感提供情绪出口，用户为"被理解"的幻觉买单
- 变现模式：订阅制，周费 6.99 美元 ~ 终身 109.99 美元

### 1.2 项目目标

MVP 阶段目标：
- 验证用户对 AI 玄学分析的需求
- 积累用户基础，建立分享裂变机制
- 通过激励视频广告实现初步变现

## 二、技术架构

### 2.1 技术栈选择

| 模块 | 技术方案 | 说明 |

## 四、AI 模块

### 4.1 AI 服务商选择

**推荐：智谱 AI（国内首选）**
- 注册地址：https://open.bigmodel.cn/
- 优势：国内访问稳定，价格低廉
- 模型：GLM-4，按 token 计费

**备选：OpenAI**
- 需要翻墙
- 模型：GPT-4o

### 4.2 提示词工程

提示词是产品的核心竞争力，决定了 AI 输出的质量和用户体验。

**设计原则：**
1. 温柔浪漫，避免冷硬措辞
2. 多用比喻，让描述有画面感
3. 保持积极正向，避免负面暗示
4. 所有结果加"仅供参考"声明
5. 使用确定性语言时加"可能/倾向/参考"

**提示词文件位置：** `prompts/bazi-prompt.md`

### 4.3 AI 调用流程

```
用户输入 → 云函数 → 提示词模板 → AI API → 解析JSON → 返回结果
                ↓
          保存到云数据库
```

## 六、变现方案

### 6.1 激励视频广告

**逻辑流程：**
```
用户免费次数用完
       ↓
弹窗提示"观看视频获取更多次数"
       ↓
用户点击 → 加载广告 → 用户观看
       ↓
广告完成 → +1 次生成机会
       ↓
广告收益归开发者
```

**代码实现：**
```javascript
// 激励视频广告
let rewardedVideoAd = wx.createRewardedVideoAd({
  adUnitId: 'YOUR_AD_UNIT_ID'
});

rewardedVideoAd.onRewarded(() => {
  // 增加免费次数
  const todayUses = Math.max(0, (wx.getStorageSync('todayUses') || 0) - 1);
  wx.setStorageSync('todayUses', todayUses);
  wx.showToast({ title: '获得1次分析机会！' });
});
```

### 6.2 后续变现方向

## 七、合规注意事项

### 7.1 内容合规

1. 所有结果页面必须加免责声明："本结果仅供娱乐参考，请理性看待"
2. 避免使用确定性语言（"你一定会..."），用"可能/倾向/参考"
3. 玄学内容不得涉及封建迷信定性，仅作为娱乐产品
4. AI 输出需过滤敏感内容

### 7.2 微信审核注意

1. 小程序类目选择"工具"或"娱乐"
2. 玄学类内容需加年龄限制提示
3. 不得声称可以预测具体事件（只能娱乐参考）
4. 隐私政策必须明确告知数据使用方式

## 九、注意事项

### 9.1 AI API 配置

1. 智谱 AI 注册：https://open.bigmodel.cn/
2. 在微信云开发控制台设置环境变量 `ZHIPU_API_KEY`
3. 当前代码内置模拟数据，未配置 API 前可正常测试 UI

1. 在微信公众平台申请广告位

1. 在微信开发者工具中，右键 `cloudfunctions` 文件夹
2. 选择"上传并部署"
3. 依次部署 `login` 和 `ai-analyzer`
