# AI 玄学 + 灵魂伴侣小程序 - 项目规格书

## 1. 项目概述

- **项目名称**：AI Fortune（AI 命理）
- **项目类型**：微信小程序（uni-app）
- **核心功能**：AI 玄学分析 + 灵魂伴侣解读，通过精美的卡牌形式展示结果，用户可分享至微信群/朋友圈
- **目标用户**：18-35 岁女性为主，对玄学/星座/塔罗感兴趣的用户
- **变现模式**：激励视频广告（微信 ad 组件），暂不接支付

---

## 2. 技术栈

| 模块 | 技术方案 |
|------|---------|
| 前端框架 | uni-app + Vue3（微信小程序） |
| 后端 | 微信云开发（免服务器） |
| AI 分析 | GPT-4o / 智谱 GLM-4（按 token 计费） |
| 数据库 | 微信云数据库 |
| 文件存储 | 微信云存储 |
| 广告 | 微信激励视频 ad 组件 |
| 支付 | 暂不接入 |

---

### 3.1 核心页面

## 4. AI 提示词规格

### 4.1 系统角色

```
你是一位资深玄学分析师，擅长八字命理、星盘解读、塔罗占卜。
用温柔、浪漫、富有诗意的语言风格输出。
```

### 4.2 分析维度

**八字版：**
- 性格特点（3个核心特质）
- 爱情观（优势和盲点）
- 理想伴侣特征（外貌+性格+相处模式）
- 正缘出现的时机/信号
- 给用户的爱情建议

**星盘版：**
- 太阳星座性格解读
- 月亮星座情感模式
- 上升星座外在表现
- 金星星座爱情风格
- 理想伴侣类型

**塔罗版：**
- 当前情感状态牌阵
- 核心挑战
- 建议行动
- 未来3个月感情走向

### 4.3 输出格式（JSON）

```json
{
  "title": "你的命定之人",
  "personality": { "tags": [], "description": "" },
  "loveStyle": { "strength": "", "weakness": "" },
  "idealPartner": { "appearance": "", "personality": "", "scenario": "" },
  "timing": { "when": "", "signs": [] },
  "advice": ["", "", ""],
  "tags": ["温柔", "专一", "事业型"]
}
```

### 4.4 语气要求

- 温柔浪漫，避免冷硬措辞
- 多用比喻，让描述有画面感
- 保持积极正向，避免负面暗示
- 免责声明："本结果仅供参考娱乐"

## 6. UI/UX 规格

### 6.1 视觉风格

- **风格**：神秘浪漫 + 现代简约
- **主色调**：深紫色 + 金色 + 白色
- **字体**：系统默认字体，卡片标题用粗体
- **动画**：柔和的渐变和悬浮效果

### 6.2 页面结构

```
首页
├── 顶部：App 名称 + 副标题
├── 中部：玄学类型选择（三个卡片）
├── 表单区：日期选择器 + 性别选择
└── 底部：开始分析按钮 + 免责声明

分析结果页
├── 顶部：结果标题（如"你的命定之人"）
├── 性格卡牌：标签 + 描述文字
├── 爱情观卡牌：优势 + 盲点
├── 理想伴侣卡牌：外貌 + 性格 + 场景描述
├── 正缘时机卡牌：时间 + 信号
├── 建议列表：3条建议
└── 底部：分享按钮 + 再算一次 + 看广告获取次数

个人中心
├── 用户信息区
├── 剩余次数显示
├── 历史记录列表
└── 关于/帮助
```

## 7. 变现逻辑

```
用户进入 → 每天免费 3 次生成
       ↓
次数用完 → 弹窗："观看视频获取更多次数"
       ↓
用户看完激励视频 → +1 次生成机会
       ↓
广告收益归开发者（前期量少，以积累用户为主）
```

---

## 8. 合规注意事项

1. 所有结果页面必须加免责声明："本结果仅供娱乐参考"
2. 避免使用确定性语言（"你一定会..."），用"可能/倾向/参考"
3. 玄学内容不得涉及封建迷信定性，仅作为娱乐产品
4. 用户数据仅存储在微信云开发环境，符合微信安全要求
